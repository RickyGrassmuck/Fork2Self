Fork repos into your own Git server.
One click. Zero ceremony.

Fork2Self is a Firefox extension that mirrors any GitHub, GitLab, Gitea, Forgejo, or Bitbucket repo into a self-hosted instance you actually control. Because "someone else's computer" is also a fine description of every Git host that isn't yours.

How it works

A three-step process, much like every onboarding flow ever shipped, but this one is real.

Configure a destination

Point it at your self-hosted Git instance and paste a personal access token. Test the connection. Be quietly proud of yourself.

Open a repo page

GitHub, GitLab, Gitea, Forgejo, Codeberg, Bitbucket — anywhere with a recognizable repo URL. The extension does the parsing.

Click "Fork"

Wait a few seconds. Your destination clones the repo (and optionally the issues, PRs, wiki, and so on). Done.

Install

The extension isn't on AMO yet because the human running this project hasn't gotten around to it. You have options.

Option 1 — Build it yourself

git clone https://git.ladon-cloud.ts.net/joyrida/Fork2Self
cd Fork2Self
bun install
bun run package
# fork2self-X.Y.Z.xpi appears at the repo root

Then in Firefox: about:debugging → This Firefox → Load Temporary Add-on… → pick dist/manifest.json.

Option 2 — Run the CI workflow

The repo has a Gitea Actions workflow that builds, lints, signs (via web-ext), and uploads an artifact. It's gated to manual runs only, because nobody needs a CI bill from a hobby project.

Option 3 — Wait

If a signed XPI appears on a release page somewhere, that was probably a future version of this project being slightly more responsible. Check the repo.

FAQ

Anticipated questions. Some of them rhetorical.

Did an AI really write all of this?

Yes. The TypeScript, the build pipeline, the README, the Gitea workflow, the Taskfile, this homepage — including this very FAQ entry confirming that an AI wrote everything. The recursion is intentional.

Is the code any good?

It typechecks under --strict, lints clean with web-ext lint (zero warnings), and survived a security review. Make of that what you will.

Why "Fork2Self"?

Because "fork yourself" is funnier in a directory listing than as a command-line invocation, and because every other name was already taken on npm.

Does it phone home?

No. The manifest declares data_collection_permissions: ["none"], the only network calls are to your configured destination and the source you're forking from, and there is no analytics SDK lurking in the dependencies. We don't even know you're using it. We can't.

Why Firefox only?

Manifest V3 portability across browsers is, charitably, a saga. Adding Chrome support is left as an exercise for whoever forks this repo into their self-hosted Git instance using the extension itself.
